Exception Map · Lab

happy-path map(/map)에 빠진 한 축 — 예외(unhappy path)를 어떻게 그릴지 세 가지 안을 같은 데이터로 비교한다. 예외는 실제 raise 지점에서 AST로 추적한다(발생하지 않는 예외는 싣지 않음).

A · 응답 카탈로그 B · 레이어 exit-ramp C · 커버리지 매트릭스
4xx · ClientError (사용자 노출) 5xx · DevelopError (prod 마스킹)
A

응답 카탈로그 — 클라이언트가 보는 것

엔드포인트마다 가능한 HTTP 응답을 한 카드에 모은다. OpenAPI responses처럼 200 + 모든 4xx/5xx를 코드별로. + "이 API가 뭘 던지나"가 한눈에 · − 어느 레이어에서 나는지는 안 보임

B

레이어 exit-ramp — 어디서 터지나

happy-path 흐름(endpoint → usecase → domain → db)을 가로로 두고, 각 레이어에서 예외가 아래로 빠져나가는 exit-ramp로 그린다. + 발생 위치가 명확, 기존 map과 일관 · − 카드 밀도 증가

C

커버리지 매트릭스 — 전수 점검

엔드포인트(행) × HTTP 코드(열). 어떤 API가 어떤 상태를 낼 수 있는지 격자로. + 빠진 곳·치우침이 보인다(예: 401 누락) · − 구체 예외명은 hover로
